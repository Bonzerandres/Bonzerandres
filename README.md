# Hi, I'm Andres Del Orbe 👋

I'm a Solutions Architect/Data Engineer focused on building practical data platforms, automated pipelines, CI/CD workflows, and cloud infrastructure that solve real operational problems.

My work sits between **data engineering, analytics engineering, cloud infrastructure, FinOps, and DevOps delivery**. I care about making systems that are useful, explainable, cost-conscious, and reliable enough to support real decisions instead of just showing off tools.

A lot of my projects start from the same place: messy data, manual workflows, disconnected systems, and teams relying on spreadsheets, exports, or one-off scripts. ## Main Projects

### Recession Signal Monitoring Platform
Built a serverless analytics pipeline that tracks U.S. recession risk using multiple FRED economic indicators instead of relying on one delayed signal. The project pulls macroeconomic data, transforms it into analysis-ready models, scores recession conditions, and prepares the output for dashboarding and backtesting.

The goal was not to overbuild a big data system. The goal was to create a clean, cost-conscious data workflow that could update consistently, explain its logic, and support quantitative financial analysis.

### Agentic FinOps Intelligence Platform
Building a cost-intelligence platform that turns cloud spend, usage, and infrastructure data into analytics-ready models and AI-assisted recommendations.

The project is focused on the data engineering side of FinOps: collecting infrastructure and billing data, modeling cost drivers, identifying waste, and creating a foundation for automated recommendations without losing visibility into how decisions are made.

### ATR Platform Unification
Designed a modernization plan for a company operating across fragmented CRM, ERP, warehouse, support, e-commerce, and reporting systems.

The work focuses on replacing disconnected tools and manual workflows with a unified operational data platform. The goal is to improve visibility across sales, warehouse operations, customer support, production, reporting, and recurring service workflows.

### Scarlet Visitor Management Platform
Built a security-focused visitor and building-access platform with identity validation, access controls, data integrity, and CI/CD delivery workflows.

This project combines application architecture, data integrity, and delivery automation. The focus is on making access workflows traceable, controlled, and easier to manage from development through deployment.

## How I Think About Engineering

I do not believe good engineering is just listing every tool possible. The important part is understanding the tradeoff behind the tool.

For small analytical workloads, I would rather use a lightweight stack that is fast, simple, and cost-effective than force a distributed platform where it is not needed. For larger systems, I care about reliability, observability, version control, and the ability to explain how data moved from raw input to final output.

The work I enjoy most is where data engineering, cloud infrastructure, and business operations meet: taking something messy, making it measurable, and building the pipeline that lets people trust the result.
